How It Works

Describe your bot. Kaida does the rest.

No coding needed. Tell Kaida what your bot should do in plain English. Kaida generates the safety rules and watches your bot in real time.

1

Install your bot first

Set up your AI agent as you normally would — OpenClaw, LangChain, CrewAI, or any framework. Make sure it works on its own. Kaida doesn't replace your bot — it protects it.

2

Install Kaida Shield

One command. Works on Windows, Linux, and macOS.

pip install kaida-shield
3

Describe what your bot should do

Open the dashboard and tell Kaida in plain English: "My bot scrapes job listings from LinkedIn and Indeed, saves them to a spreadsheet in my Documents folder. It should not access any other websites or write files anywhere else." Kaida generates the safety rules automatically.

kaida ui
4

Run your bot with protection

Start your bot through Kaida — in the dashboard or from the terminal. Kaida monitors which websites it connects to, which files it touches, and what commands it runs. If everything matches your rules, you'll never notice Kaida is there.

kaida shield run --policy my-policy python my_bot.py
5

Kaida freezes, not kills

If your bot tries something outside your rules, Kaida freezes it — doesn't kill it. Your work is preserved. You decide what to do: release it, adjust the rules, or stop it. No work lost.

Transparency

What Kaida does — and doesn't do

We believe in honesty. Here's exactly where Kaida protects you today, and what's still on the roadmap.

✓ Works today

  • Freezes bots accessing unauthorized websites
  • Catches unauthorized file read/write attempts
  • Detects phishing URLs and brand impersonation
  • Blocks reverse shells, crypto miners, malicious commands
  • Catches prompt injection attempts
  • Detects bots stuck in loops
  • Alerts on excessive API calls and token burn
  • Alerts on activity spikes
  • Notifies when bots go idle
  • Natural language policy creation — no coding needed
  • Works on Windows, Linux, and macOS
  • Works with any agent framework

→ Limitations & roadmap

  • Cannot see inside encrypted traffic (sees destinations, not content)
  • Cannot monitor what the AI model is "thinking"
  • macOS-specific deep monitoring (in development)
  • Cloud-based dashboard (currently runs locally — coming soon)
  • Mobile push notifications via Telegram (in beta)
